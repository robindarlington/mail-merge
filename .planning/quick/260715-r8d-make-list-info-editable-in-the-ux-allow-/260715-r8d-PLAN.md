<objective>
Make an uploaded list's display name editable in the UX. A signed-in user can rename
any of their own lists — inline from the Lists page row and from the list detail
header — for clarity and ease of human use.

Purpose: uploaded lists currently show only the raw CSV filename (e.g.
`PETR-EMAIL (final)(2).csv`), which is noise. Letting the user assign a friendly
name makes the Lists surface human-usable.

Output: a nullable `label` column on `recipient_sets` (mirroring the 06.1
`smtp_configs.label` precedent), an owner-scoped `renameRecipientSet` DAL function,
a validated `renameList` server action, and an inline-edit island wired into both
the Lists page and the detail header. Everywhere a list name is shown becomes
`label ?? filename`, so the original filename is preserved for reference.
</objective>

  returns { ok:false, error:{ kind:"unauthenticated" } } when no userId, then
  delegates to a userId-accepting core in lib/csv/actions-core.ts. ActionError is a
  discriminated union re-exported (type-only) from actions.ts.

ActionError union (lib/csv/actions-core.ts) already has the kinds you need:
  "unauthenticated" | "validation" (issues) | "unknown" (raw string). Reuse it.
