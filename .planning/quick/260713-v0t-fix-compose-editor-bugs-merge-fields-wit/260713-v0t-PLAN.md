<objective>
Fix two compose-editor bugs:

1. Merge fields whose column names contain spaces (e.g. `First Name`) never render — the token grammar `[\w.-]+` cannot match a space, so `{{First Name}}` passes through as literal text in preview and in sent email, and the `{{`-autocomplete popover closes the instant the user types a space.

2. The suggestion popover renders far below the editor (anchored to the "Merge fields" chip row) instead of at the caret where the user is typing.

Purpose: Merge fields with spaces are common in real CSVs (title-case headers), and a popover disconnected from the caret is confusing. Both are core to the compose slice (EDIT-02 autocomplete, EDIT-03 substitution).

Output: Widened token grammar + tests (Bug 1), and a caret-following popover via a hand-rolled mirror-div measurement with zero new dependencies (Bug 2).
</objective>
