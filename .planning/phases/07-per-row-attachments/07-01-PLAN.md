<objective>
Lay the persistence + safety foundation for per-row attachments: migrate the `attachments` table so an upload can exist BEFORE a campaign does (owner column, nullable campaign/send_record FKs, a persisted `size_bytes`), add the traversal-proof storage writer/resolver, centralize the size-limit constants, and add the filename-column auto-detector. Everything downstream (DAL, actions, confirm gate, worker, UI) composes over these primitives.

Purpose: Attachments are uploaded on `/compose` long before the campaign row is created (`prepareCampaignCore`), so the committed table shape (both FKs NOT NULL) cannot hold them. This plan re-keys the table and installs the opaque-id storage discipline that makes attachment resolution traversal-safe (ATCH-03) — the security spine of the whole phase.
Output: migration `0006`, `lib/attachments/{storage,schema}.ts` (+ storage test), `lib/core/attachment-column.ts` (+ test), all green under the test runner.
</objective>

## Phase Goal (user story)

**As a** signed-in sender, **I want to** attach a different file to each row of my CSV — with every referenced file validated as present and resolved safely — **so that** I can send personalized emails that each carry the right per-recipient file with confidence nothing is missing or unsafe.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client filename → disk path | The user-supplied original filename must NEVER become a path component (traversal) |
| stored storage_path → filesystem read | A path from the DB is resolved+prefix-checked before any disk access |
