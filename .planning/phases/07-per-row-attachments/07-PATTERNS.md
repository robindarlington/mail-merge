## CRITICAL SCHEMA FINDING (read first)

The `attachments` table EXISTS in `lib/db/schema.ts` (lines 177-188) with EXACTLY these columns — plans must not invent others:

```typescript
// lib/db/schema.ts:177-188 — the CURRENT committed shape
export const attachments = sqliteTable("attachments", {
  id: integer("id").primaryKey({ autoIncrement: true }),
  campaign_id: integer("campaign_id")
    .notNull()
    .references(() => campaigns.id),
  send_record_id: integer("send_record_id")
    .notNull()
    .references(() => send_records.id),
  filename: text("filename").notNull(),
  storage_path: text("storage_path").notNull(),
  created_at: integer("created_at").notNull().default(unixNow),
});
// Typed models already exported: Attachment / NewAttachment (schema.ts:207-208)
```

**A migration IS needed.** Three concrete conflicts between this shape and the CONTEXT.md decisions:

1. **`send_record_id` is NOT NULL** — but send_records only exist after the worker claims the campaign and runs `materializeSendRecords` (lib/worker/materialize.ts). Attachments are uploaded on `/compose`, long before any send_record exists. Either `send_record_id` must become nullable (stamped later at materialize time, mirroring how `recipient_sets.email_column` was added nullable-additively) or be dropped in favor of resolving by filename at materialize time.
2. **`campaign_id` is NOT NULL** — but campaigns are created at the "review and send" moment (`prepareCampaignCore`, lib/campaign/actions-core.ts:342), AFTER compose. If uploads happen before prepare (as CONTEXT decides: "attachment upload lives on the compose page... gated after a recipient list is selected"), the row cannot carry a campaign FK at insert. Options: (a) add `userId` + nullable `recipient_set_id`, make `campaign_id` nullable and stamp it at prepare; (b) keep campaign_id NOT NULL and restructure the UI flow to upload after prepare. Option (a) matches the decided UX. NOTE: tenancy-via-campaign-FK (AUTH-02 comment in schema.ts:16-18) only works once campaign_id is set — the DAL needs a direct owner scope for the pre-campaign window.
3. **No `size_bytes` column** — the 10 MB per-file / 15 MB per-message validation sums sizes at confirm time; a persisted size column avoids stat-ing disk per row (and survives a missing file gracefully). Additive NOT NULL-with-no-default is fine only on a recreated table; otherwise nullable.

SQLite cannot ALTER a column's nullability — drizzle-kit will generate a table-recreate migration for change 1/2 (fine: the table has zero production rows; verify with a data check in the plan). The additive-column analog is `drizzle/0005_list_label.sql` (`ALTER TABLE recipient_sets ADD label text;` — one line, generated by `npx drizzle-kit generate`).

**Second body-size finding:** `next.config.ts:13` sets `experimental: { serverActions: { bodySizeLimit: "4mb" } }` and `lib/csv/schema.ts:21` keeps `MAX_UPLOAD_BYTES = 4MB` "in step with the Server Action bodySizeLimit". A 10 MB attachment CANNOT pass through a server action today. The plan must either bump `bodySizeLimit` (e.g. `"11mb"` to fit one 10 MB file + form overhead, uploading files ONE per action call) or route uploads through a route handler. Uploading one-file-per-action-call is the smallest delta and keeps the csv analog intact.

## Pattern Assignments

### `lib/attachments/storage.ts` (storage writer, file-I/O)

**Analog:** `lib/csv/storage.ts` (57 lines — copy the whole module shape)

**Env-dir resolution + opaque-UUID write** (lines 25-36):
```typescript
const UPLOADS_DIR = resolve(process.env.UPLOADS_PATH ?? "./data/uploads");

export function writeUpload(bytes: Buffer): { storagePath: string } {
  mkdirSync(UPLOADS_DIR, { recursive: true });
  const name = `${randomUUID()}.csv`; // opaque — user filename never in the path
  writeFileSync(resolve(UPLOADS_DIR, name), bytes);
  return { storagePath: name }; // store RELATIVE; resolve at read time (Pitfall 4)
}
```
For attachments: same `UPLOADS_PATH` volume (CONTEXT decision), either a `attachments/` subdir or a different extension scheme (`${randomUUID()}.bin` — original extension must NOT be trusted into the path; keep the original filename in the DB only). Keep the returned path RELATIVE.

**Traversal-checked read** (lines 50-56) — the worker needs this to resolve at send time:
```typescript
export function readUpload(storagePath: string): Buffer {
  const full = resolve(UPLOADS_DIR, storagePath);
  if (full !== UPLOADS_DIR && !full.startsWith(UPLOADS_DIR + sep)) {
    throw new Error("resolved upload path escaped the uploads directory");
  }
  return readFileSync(full);
}
```
The worker will pass a resolved ABSOLUTE path to nodemailer's `attachments: [{ path }]` — expose a `resolveAttachmentPath(storagePath): string` (same prefix check, returns the absolute path instead of bytes) plus an existence check for the presence validation, rather than reading bytes into memory.

  { ok: true; file: File } | { ok: false; error: ActionError } {
  const file = formData.get("file");

}

## Shared Patterns

### actions / actions-core split (endpoint safety)
**Source:** header comments of `lib/csv/actions-core.ts:1-24`, `lib/compose/actions-core.ts:1-31`, `lib/campaign/actions-core.ts:1-36`
**Apply to:** `lib/attachments/actions.ts` + `actions-core.ts`
The `"use server"` module exports ONLY auth-wrapping actions (userId re-derived via lazy-imported Clerk `auth()`); every userId-accepting seam lives in a directive-free `actions-core.ts`. Type re-exports from the action module are type-only.

### Closed ActionError union, string-only raw
**Source:** `lib/csv/actions-core.ts:57-65` (and per-module variants)
**Apply to:** all new seams. `{ ok: true; data } | { ok: false; error: ActionError }`, never rejects; `raw: String((e as Error)?.message ?? e)`.

### userId-scoped DAL, owner-filter grep gate
**Source:** `lib/data/recipients.ts` (whole module)
**Apply to:** `lib/data/attachments.ts`. userId FIRST param on every function; `and(eq(id), eq(userId))` on every by-id path; `Pick<>` omitting userId + spread-userId-LAST on create; inline "owner-filter, AUTH-02 grep gate" comments.

### Worker tenancy exception
**Source:** `lib/worker/materialize.ts:22-25`, `process.ts:30-34`
**Apply to:** worker attachment resolution. No Clerk session — derive owner from `campaign.userId` and resolve the campaign's OWN FKs through the userId-scoped DAL.

### Temp-everything test harness
**Source:** `lib/data/recipients.test.ts:21-33`, `lib/worker/process.test.ts:26-46`, `lib/csv/storage.test.ts:20-29`
**Apply to:** every new test file. Env vars (`DATABASE_PATH`, `UPLOADS_PATH`, `CREDENTIAL_ENC_KEY`) BEFORE dynamic `await import(...)`; `migrate` from `./drizzle`; `connection.close()` + `rmSync` in `after()`.

### better-sqlite3 synchronous-transaction rule
**Source:** `lib/worker/process.ts:22-24, 196-252`
