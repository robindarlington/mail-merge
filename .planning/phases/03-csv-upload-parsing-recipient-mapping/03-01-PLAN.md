Consumed (in-repo, proven):
- `lib/core/csv.ts` already exports `type Row = Record<string, string>`, `interface ParsedCsv`, `function parseCsv(input: string | Buffer): ParsedCsv`, and a module-level `EMAIL_RE = /^[^\s@]+@[^\s@]+\.[^\s@]+$/`. REUSE `EMAIL_RE` and `Row` — do NOT redeclare either.

Defined (consumed by 03-03 actions-core):
- `detectEmailColumn(columns: string[], rows: Row[]): string | null` — two-stage: normalized header-name match against `["email","e-mail","mail","email address","recipient"]` (exact/normalized preferred, then `.includes("email")`), else content-sampling over `rows.slice(0,50)` scoring each column by EMAIL_RE hit-rate on non-empty values; return the best scorer only if `score > 0.7`, else null. Full reference impl: 03-RESEARCH.md lines 271-289.
- `countInvalidEmails(rows: Row[], column: string): number` — counts rows whose `(row[column] ?? "").trim()` fails EMAIL_RE. Reference impl: 03-RESEARCH.md lines 291-296.
- `writeUpload(bytes: Buffer): { storagePath: string }` — mkdir `UPLOADS_DIR` recursive, write to `resolve(UPLOADS_DIR, \`${randomUUID()}.csv\`)`, return `{ storagePath: "<uuid>.csv" }` (relative). Reference impl: 03-RESEARCH.md lines 193-198.
- `lib/csv/schema.ts`: `MAX_UPLOAD_BYTES = 4 * 1024 * 1024`, `MAX_ROWS = 5000`, `confirmColumnSchema = z.object({ emailColumn: z.string().min(1, "Choose the email column") })`, plus an upload-file guard validating `.csv`/`text/csv` extension+mime and `size <= MAX_UPLOAD_BYTES`. Export each schema object + its `z.infer` type.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user-supplied CSV bytes + filename → filesystem | Untrusted; only opaque bytes are written, never a user-controlled path component |
| user-supplied file metadata (name/type/size) → zod guard | Untrusted; validated before any persistence |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3-TRAV | Tampering | lib/csv/storage.ts writeUpload | mitigate | On-disk name is `crypto.randomUUID() + ".csv"`; the user filename never enters `resolve(UPLOADS_DIR, name)`; `storage_path` stored relative. Test asserts the filename is absent from the returned path (V12). |
| T-3-DOS | Denial of service | lib/csv/schema.ts upload guard | mitigate | `MAX_UPLOAD_BYTES` (4 MB) enforced in the zod guard so oversized files reject with a clear message before any parse/write; `MAX_ROWS` constant published here for the row-cap check enforced in 03-03. |
| T-3-SLOP | Tampering (supply chain) | dependencies | accept | No new npm packages installed this plan (papaparse/zod already vetted in Phase 1/2); nothing to gate. |
</threat_model>
