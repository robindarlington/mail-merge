<objective>
Let the user choose which verified SMTP server a campaign sends through, stamp that
choice on the campaign, and make every send path resolve the config by id
(owner-scoped) — closing the "wrong server / arbitrary row" gap and defining the
Phase 6 worker load contract.

## Phase Goal

**As a** signed-in user with several verified SMTP servers, **I want to** pick which
server a given campaign sends through (auto-selected when I only have one), **so that**
each campaign is delivered over the server I intend, with the choice recorded on the
campaign.

Purpose: Thread `smtpConfigId` through prepare + test-send, resolve confirm-summary
by the stamped FK, and render the compose picker with the zero-click single-server
case (SC2). Server always re-resolves the client-proposed id (SC3 IDOR).

Output: `smtpConfigIdSchema`, reworked campaign action seams + wrappers, the
`SendCard` picker + its client wiring, and a doc-level amendment to Phase 6's
06-02 plan so its executor loads by `campaign.smtp_config_id`.
</objective>
