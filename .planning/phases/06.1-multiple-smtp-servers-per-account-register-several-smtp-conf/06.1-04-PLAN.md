<objective>
Turn the single-server settings surface into a multi-server experience: a per-server
list with add/edit/verify/set-default/soft-delete, the label field and WR-09
host-change gate in the reused wizard, a destructive delete confirmation with the
in-use guard error, the no-default choose-a-server state, and updated readiness copy.

## Phase Goal

**As a** signed-in user, **I want to** see and manage all my SMTP servers on the
settings page (add new ones, set a default, delete unused ones), **so that** I can
maintain several sending identities and control which is the default.

Purpose: Deliver the visible half of the multi-server capability (SC1 verified
status per server, SC5 delete/no-default UX). Reuses the existing verify-before-save
wizard per server; adds exactly one official shadcn component (alert-dialog) and
zero npm packages.

Output: server-list.tsx client island, wizard label + WR-09 gate, the settings RSC
listing configs, the dashboard readiness copy, and an end-of-phase human-verify
checkpoint for the multi-server walkthrough.
</objective>

| RSC -> client island | only redacted DTOs (no password triple) cross to the browser |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-061-12 | Information Disclosure | settings RSC -> ServerList | mitigate | RSC maps rows through toSmtpConfigDto; the encrypted triple is never referenced, so it cannot cross to the client island |
| T-061-13 | Elevation / IDOR | setDefaultServer / deleteServer from the list | mitigate | Actions (Plan 02) re-derive userId and resolve the id owner-scoped; the client only sends an id |
| T-061-14 | Denial of Service / Repudiation | delete of an in-use server | mitigate | in_use ActionError keeps the row; dialog surfaces the actionable message; soft-delete never runs for a queued/running-referenced server |
| T-061-15 | Information Disclosure (WR-09) | wizard edit with changed host | mitigate | Client gate blocks blank-keep on host change; server re-checks (Plan 02) - defense in depth |
| T-061-SC | Tampering | npm / shadcn installs | mitigate | Only the official shadcn alert-dialog is added (no npm dependency); package.json unchanged, asserted in Task 1 acceptance |
