### `drizzle/0003_*.sql` + `drizzle/0004_backfill_default_smtp.sql` (migration)

**Analog:** `drizzle/0001_shiny_stature.sql` (a standalone `CREATE UNIQUE INDEX` — proves the drop is metadata-only) and `drizzle/0002_nappy_sabretooth.sql` (`ALTER TABLE ... ADD` single statement).

`0001` in full (the index being dropped):
```sql
CREATE UNIQUE INDEX `smtp_configs_user_uq` ON `smtp_configs` (`user_id`);
```
`0002` in full (the ADD-COLUMN idiom to expect ×3):
```sql
ALTER TABLE `recipient_sets` ADD `email_column` text;
```

**Do NOT hand-write `0003`** — generate it via `npm run db:generate`, then **read the file** and assert it is `DROP INDEX` + `ALTER ADD` (×3) + `CREATE UNIQUE INDEX ... WHERE`, NOT `CREATE TABLE __new_smtp_configs` (RESEARCH Pitfall 1 verification gate). Expected shape:
```sql
DROP INDEX IF EXISTS `smtp_configs_user_uq`;--> statement-breakpoint
ALTER TABLE `smtp_configs` ADD `label` text;--> statement-breakpoint
ALTER TABLE `smtp_configs` ADD `is_default` integer DEFAULT 0 NOT NULL;--> statement-breakpoint
ALTER TABLE `smtp_configs` ADD `deleted_at` integer;--> statement-breakpoint
CREATE UNIQUE INDEX `smtp_configs_user_default_uq` ON `smtp_configs` (`user_id`) WHERE `is_default` = 1;
```

`0004` custom data migration (hand-authored, tracked in `_journal.json`) — mirrors the single-statement style of `0002`:
```sql
UPDATE `smtp_configs` SET `label` = COALESCE(`label`, 'Default'), `is_default` = 1 WHERE `deleted_at` IS NULL;
```
> Order matters: columns → backfill → partial index. Since pre-migration there is ≤1 row/user, unconditional `is_default = 1` cannot violate the partial index.

## Shared Patterns

### IDOR-safe owner-scoped access (V4 Access Control — PRIMARY)
**Source:** `lib/data/campaigns.ts:68` (`getCampaignForUser`), `lib/data/recipients.ts:67`
**Apply to:** every new `lib/data/smtp.ts` by-id/list/update/delete fn; every send-path config load.
```typescript
where: and(eq(smtp_configs.id, id), eq(smtp_configs.userId, userId), isNull(smtp_configs.deleted_at))
```
There is NO fetch-by-id-alone path. A client `smtpConfigId` is always re-resolved server-side; a cross-tenant/unknown/deleted id becomes `not_found`/`undefined`.

### Server-injected ownership (T-TAMPER-OWNER)
**Source:** `lib/data/campaigns.ts:55-60` (`createDraftCampaign`), `recipients.ts:42-50`
**Apply to:** `createSmtpConfig`.
```typescript
db.insert(smtp_configs).values({ ...values, userId }).returning();  // userId LAST — ownership wins
```
The `values` type is a `Pick<>` that OMITS `userId` (`smtp.ts:43` `PersistableConfig`) so a caller cannot spoof it.

### Affected-row-count as the transition signal (atomic, no TOCTOU)
**Source:** `lib/data/campaigns.ts:83-95` (`enqueueCampaign`)
**Apply to:** `updateSmtpConfigById`, `softDeleteSmtpConfig`, `setDefaultSmtpConfig`.
```typescript
.returning({ id: smtp_configs.id });  // length 1 = success, 0 = IDOR/not-found/no-op
```
Never SELECT-then-UPDATE. For the default-swap, wrap clear-all + set-one in `db.transaction()`.

### Single server→client redaction boundary (V6 Cryptography / SMTP-04)
**Source:** `lib/data/smtp.ts:122-132` (`toSmtpConfigDto`)
**Apply to:** every RSC/action returning config data (settings, compose, dashboard, confirm-summary).
Keep EXPLICIT field enumeration; adding `id/label/is_default` must not become a row spread. `password_enc/_iv/_tag` are absent BY CONSTRUCTION, verified by `smtp.test.ts:160-167`.

### "use server" auth preamble (V2 Authentication / AUTH-02)
**Source:** `lib/smtp/actions.ts:63-69`, `lib/campaign/actions.ts:64-66`
**Apply to:** every new Server Action.
```typescript
const { auth } = await import("@clerk/nextjs/server");   // lazy — keeps module test-loadable
const { userId } = await auth();
if (!userId) return { ok: false, error: { kind: "unauthenticated" } };
```
