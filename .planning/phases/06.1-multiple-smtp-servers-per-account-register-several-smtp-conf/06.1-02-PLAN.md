<objective>
Give the server the ability to register, verify, label, re-authenticate, set-default,
and soft-delete MANY SMTP configs per user — the backend for the multi-server
settings surface.

## Phase Goal

**As a** signed-in user, **I want to** register and manage several named SMTP servers
on my account (each verified and encrypted independently), **so that** I can pick
which one a campaign sends through.

Purpose: Split the retired verify-and-upsert into explicit create (insert) and
update-by-id paths, close the WR-09 credential-exfil path (password re-entry on
host change, LOCKED), and enforce the soft-delete in-use guard (SC5). No new npm
packages.

Output: `label` on the shared schema, id-scoped action seams in actions-core, the
thin `"use server"` wrappers in actions.ts, and extended action tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → "use server" action | client supplies a config `id`, a possibly-changed `host`, and a blank password |
| server → user's BYO SMTP host | a stored credential must never be dialed at a client-changed host |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-061-05 | Information Disclosure (WR-09) | applyVerifiedConfig blank-keep merge | mitigate | Host-equality check BEFORE decrypt; blank+changed-host rejected with a validation issue; stored credential never sent to the new host (LOCKED CONTEXT decision) |
| T-061-06 | Elevation / IDOR | setDefaultServer / deleteServer / updateServer / sendTestEmail | mitigate | Every action re-derives userId via auth() and resolves the id through the owner-scoped DAL; 0-length result → not_found |
| T-061-07 | Denial of Service / Repudiation | deleteServer on an in-use config | mitigate | countActiveSendsForConfig gate blocks soft-delete while a queued/running campaign references the server (kind:"in_use") |
| T-061-08 | Spoofing / SSRF | createServer/updateServer host dial | mitigate | isPrivateHostLiteral SSRF screen + underVerifyRateLimit reused per add/edit |
| T-061-SC | Tampering | npm installs | accept | Zero packages installed this plan (06.1-RESEARCH audit) |
</threat_model>
