# Phase 06.1 Plan 03: Compose Server Picker + Owner-Scoped Send Paths Summary

**Users pick which verified SMTP server a campaign sends through (auto-selected when only one), the choice is stamped on the campaign at prepare, and every send path (prepare / test-send / confirm-summary) resolves the config by id owner-scoped — closing the wrong-server / arbitrary-row gap.**

## Decisions Made
- **Missing/invalid server selection is `validation`, unknown id is `not_found`:** the plan explicitly directed this split; `no_smtp_config` is no longer emitted by prepare/test-send (the union variant is retained so exhaustive UI switches keep compiling).
- **Send actions gated on a chosen server:** `TestSendPanel` and `ConfirmSendDialog` require a `number` id and render only when `smtpConfigId !== null`, so the multi-server no-default state ("Choose which server to send with above.") keeps both actions inert until a choice stands.
- **Re-prepare on server change:** `smtpConfigId` was added to the ConfirmSendDialog stale-summary guard and prepare-effect deps so a review can never render a summary/draft stamped with a previously chosen server.
