## Pattern Assignments

### `packages/cli/src/secrets.ts` (utility, request-response) — EXACT ANALOG

**Analog:** `send-credentials.ts` (`promptHidden` 65–78, `env` 58–62, pass intake 133–135)

**Port `promptHidden` verbatim** (`send-credentials.ts:65-78`) — the no-echo readline prompt. Node stdlib only (`node:readline`), no dep. Keep the `_writeToOutput` mute trick.

**Env-first, prompt-fallback intake** — the exact D-07 contract already exists at `send-credentials.ts:133-135`:
```typescript
let pass = process.env.SMTP_PASS;
if (!pass) pass = await promptHidden(`SMTP password for ${user} (input hidden): `);
if (!pass) throw new Error("No SMTP password provided.");
```
Carry the rule from `env()` (58–62): required-var-missing throws a message naming the VAR, never the value. The password is NEVER read from argv (RESEARCH Pattern 2 — deliberately absent from the parseArgs schema) and NEVER logged.

### `packages/cli/src/args.ts` (utility, transform) — ROLE-MATCH

**Analog:** `worker/index.ts::envInt` (45–57) for numeric-coercion discipline; `send-credentials.ts::main` (107–116) for the run-mode matrix.

**Fail-closed numeric coercion** — `--delay-ms` has no `number` type in `parseArgs`, so coerce like `envInt` does: reject a set-but-malformed value rather than degrade to `NaN` (which would zero the throttle). Error names the flag, never spins:
```typescript
// worker/index.ts:45-57 — the discipline to mirror for --delay-ms
const n = Number(raw);
if (!Number.isFinite(n) || n <= 0) { /* refuse, name the var only */ }
```

**Run-mode matrix** — mirror `send-credentials.ts:112-116`: `--send` (live), `--test ADDR` (whole batch to one address, error if ADDR missing), else dry-run default. `parseArgs({ strict: true })` (the default) rejects an unknown `--password`/`--smtp-pass` flag for free — this IS the SC-2 secret-safety guarantee; keep those flags out of the schema (RESEARCH Pattern 2).

### `packages/cli/src/run.ts` (service, batch) — ROLE-MATCH (the one shared driver)

**Analog:** `send-credentials.ts` send loop (146–171) + `lib/core/send.ts` (`sendOne` 140–164, `throttle` 167–170).

This is the SINGLE send-orchestration module both the CLI driver and the MCP `send`/`test-send` tools call, so the four success criteria can never diverge (RESEARCH §System Architecture, Anti-Patterns "Building the send loop twice").

**Loop shape** — carry the mode/tag/throttle structure from `send-credentials.ts:146-171` but replace the inline try/catch with the structured `sendOne` result:
```typescript
// per row: fillMessage(tpl, row) → sendOne({transport, from, to, subject, body})
// sendOne NEVER throws — it returns { ok:true, messageId } | { ok:false, error }
// (lib/core/send.ts:90-92,140-164). Serialize that result straight into a receipt line.
await throttle(delayMs); // lib/core/send.ts:167 — carry-forward of DELAY_MS (default 3000)
```
Use `createSmtpTransport` (explicit `secure` boolean — NEVER infer TLS from the port; `lib/core/send.ts:100-120`) and `verifyTransport` before the loop (the CLI's "SMTP connection OK" gate, `send-credentials.ts:142`). Preserve the `[i+1/N] sent -> addr` / `FAILED ->` log lines (160–162) and the `DRY RUN:`/`TEST mode:`/`LIVE mode:` banners (122–124).

**Resume-skip** — before the loop, build a `Set<string>` of `to` addresses whose recorded status is `sent` from `receipts.ts`, and skip them (file-based analogue of the worker's exactly-once; RESEARCH Pitfall 3 — document as at-least-once, `fsync` each line before the next send).

### `packages/cli/src/receipts.ts` (utility, file-I/O) — EXACT ANALOG

**Analog:** `scripts/stub-smtp.ts` JSONL append + read (27–71, 96–103).

**Append one line per record** (mirror `stub-smtp.ts:96-103`) — `appendFileSync(path, JSON.stringify(entry) + "\n")`; each line is `{to, status: "sent"|"failed", messageId|error, timestamp}` (D-11). Add an `fsync` after append (RESEARCH Pitfall 3) to shrink the crash-resend window.

**Read-sent-set for `--resume`** (mirror `stub-smtp.ts:64-71`) — tolerate a missing file (`existsSync` → `[]`), split on `\n`, trim, filter empties, `JSON.parse` each line; reduce to a `Set` of `to` where `status === "sent"`.

**Default path derivation (D-12):** `mydata.csv` → `mydata.receipts.jsonl` (swap/append extension off the `--csv` value); `--receipts <file>` relocates, `--no-receipts` disables. On by default for live sends only.

### `packages/cli/src/mcp.ts` (controller, request-response) — ROLE-MATCH (no MCP analog in repo)

**Analog for RESULT SHAPE:** `lib/compose/actions-core.ts` (51–92) — the typed closed-union `{ ok: true; data } | { ok: false; error }` seam. Mirror this discipline for the MCP tool outputs (`outputSchema` → `structuredContent`), and carry its logging rule: NEVER `console.*` a subject/body/CSV cell; an error `raw` field is ALWAYS a string (actions-core.ts:31, 146–147).

**Analog for SERVER WIRING:** none in repo — use RESEARCH Pattern 1 (SDK 1.29.0, VERIFIED): `McpServer` + `StdioServerTransport` + `registerTool(name, { description, inputSchema }, cb)` with a RAW zod shape (not `z.object`). Four tools: `validate-csv`, `preview-merge`, `test-send`, `send`. All delegate to `lib/core` + the shared `run.ts`. Export a `buildServer()` factory (returns the `McpServer`) so the in-process test can connect it (RESEARCH §In-process MCP test).

**Two-step confirm token (D-04):** process-local in-memory `Map` keyed to a hash of the send params; first `send` returns preview summary + one-time token and delivers nothing; second call consumes (deletes) the token and sends. Mirror the web app's preview→confirm gate (RESEARCH Pattern 3).

## Shared Patterns

### Engine reuse (the parity anchor)
**Source:** `lib/core/index.ts` (barrel, 1–35)
**Apply to:** `bin.ts`, `run.ts`, `mcp.ts`, all tests
Import every merge/CSV/send primitive from `../../lib/core` (esbuild inlines it at bundle time — the shipped artifact needs no path alias). Never re-implement `parseCsv`, `fillMessage`, `detectEmailColumn`, `isValidEmail`, or the send path (CONTEXT canonical-refs; RESEARCH "Don't Hand-Roll").

### Secret-safe logging discipline (grep-enforced carry-forward)
**Source:** `lib/core/send.ts:18-25`, `worker/index.ts:45-57` (envInt), `lib/compose/actions-core.ts:31`
**Apply to:** every new file
Error messages carry variable/flag NAMES, never values. Never log the SMTP password, the auth object, or a send body. Receipts store `to`/status/messageId/error only — never a secret. A caught-error log emits the message STRING, never the raw Error (`worker/index.ts:173,242`).

### Structured never-throws send result
**Source:** `lib/core/send.ts:89-164` (`SendResult`, `sendOne`)
**Apply to:** `run.ts`, `mcp.ts`
`sendOne` returns `{ ok:true, messageId } | { ok:false, error:{message,code?} }` and never aborts the batch — serialize it directly into a receipt line (D-11) so one bad recipient never stops the run.

### esbuild self-contained bundle
**Source:** root `package.json:14` (`build:worker`) + `.gitignore:20-21`
**Apply to:** `packages/cli/package.json` build script, root `.gitignore`
Copy the `build:worker` recipe: `esbuild <entry> --bundle --platform=node --format=esm --tsconfig=... --outfile=...`. Reuse the `--banner:js` `createRequire` shim if nodemailer's dynamic requires trip the bundle (RESEARCH Pitfall 4). Bundle the stdio-only path (~454 KB measured, express/hono tree-shaken). Add the CLI `dist` bundle to `.gitignore` exactly as `/worker.js` and `/migrate.js` are listed (lines 20–21).

### node:test harness idiom
**Source:** `lib/worker/maintenance.test.ts:22-64`, `lib/core/fill.test.ts`
**Apply to:** all `packages/cli/test/*.test.ts`
`import { test, before, after } from "node:test"` + `import assert from "node:assert/strict"`. For integration tests needing files, use `mkdtempSync(join(tmpdir(), ...))` and a FIXED clock/deterministic inputs (maintenance.test.ts:36,64). Send-path tests use `scripts/stub-smtp.ts` as the sink (no real sockets/credentials); MCP tests use `InMemoryTransport.createLinkedPair()` + `Client` (RESEARCH §In-process MCP test). Run via the project harness: `node --import tsx --test "packages/cli/test/*.test.ts"`.

## Packaging Guardrail (critical, from RESEARCH Pitfall 1 / A1)

`packages/cli` stays a STANDALONE sub-package with its OWN lockfile — do NOT add a `workspaces` key to the root `package.json` (confirmed absent this session). The Dockerfile copies only root `package.json` + `package-lock.json` then `npm ci` (`Dockerfile:30-31`, `39-40`); a workspace conversion breaks that `npm ci` and hoists the MCP-SDK tree into the production web image. The standalone layout leaves the Phase-8 deploy byte-for-byte unchanged.
