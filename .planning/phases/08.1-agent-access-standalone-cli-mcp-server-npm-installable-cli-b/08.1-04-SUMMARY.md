## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Corrected the esbuild build banner — shell-embedded `\n` did not become a newline**
- **Found during:** Task 1 (first build + run of the bundled artifact)
- **Issue:** The plan's build script set `--banner:js="#!/usr/bin/env node\nimport{createRequire…}"` as a single argument. Inside a double-quoted npm/sh script string the `\n` is a literal backslash-n, not a newline; esbuild emitted it verbatim, so `dist/bin.js` line 2 became `#!/usr/bin/env node\nimport{createRequire…}` and `node dist/bin.js` threw a SyntaxError. (The plan's own `head -1 | grep` check still passed because line 1 was a clean shebang, masking the broken line 2.)
- **Fix:** Removed the shebang from the banner. `bin.ts` already starts with `#!/usr/bin/env node`, and esbuild preserves the entry point's shebang on line 1; the banner now carries only the createRequire shim, which esbuild places on line 2. Output is now a valid two-line preamble followed by the bundle.
- **Files modified:** packages/cli/package.json (build script)
- **Verification:** `node packages/cli/dist/bin.js --help` runs; `sed -n '1,2p'` shows a clean shebang then the shim; a real `--send` through the bundle delivers to stub-smtp.
- **Committed in:** ecaaa9b (Task 1 commit)
