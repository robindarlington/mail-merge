## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| shell/argv → CLI process | Operator-supplied flags; must never be a channel for the SMTP password |
| filesystem → CLI | `--csv` / `--template` files read with the invoking user's own privileges |
| npm registry → lockfile | Third-party package (MCP SDK) enters the dependency tree |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-081-01 | Information disclosure | args.ts (argv) | mitigate | No `password`/`smtp-pass` option in the parseArgs schema; strict mode rejects unknown flags — tested in args.test.ts (SC-2) |
| T-081-03 | Tampering | template.ts / bin.ts file reads | mitigate | `resolve` + `statSync().isFile()` before read; errors name the flag, never file contents |
| T-081-05 | Input validation | args.ts numeric coercion | mitigate | `--delay-ms` rejects non-finite/≤0 (worker envInt discipline) rather than degrading to NaN |
| T-081-SC | Tampering (supply chain) | @modelcontextprotocol/sdk install | mitigate | Exact pin `@modelcontextprotocol/sdk@1.29.0`; blocking-human legitimacy checkpoint (Task 1) before install; own lockfile isolates it from the web image |
| T-081-DEPLOY | Tampering | root package.json / lockfile / Dockerfile | mitigate | Standalone sub-package (no root `workspaces`); `git diff --exit-code` on those three paths asserted in Task 2 |
