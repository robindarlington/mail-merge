# Phase 08.1 Plan 01: Standalone CLI Foundation (DRY-RUN slice) Summary

**A runnable `mail-merge --csv data.csv --template msg.txt` DRY-RUN CLI in a standalone `packages/cli` sub-package (own lockfile, pinned MCP SDK) that reuses lib/core for all CSV/merge logic behind a structurally password-safe argument contract.**

## Accomplishments
- Standalone `packages/cli` sub-package scaffolded with its OWN `package-lock.json` — the root `package.json`, root `package-lock.json`, and `Dockerfile` remain byte-identical (no root `workspaces` key), so the Phase-8 deploy surface is untouched.
- `@modelcontextprotocol/sdk` pinned to exactly `1.29.0` as a devDep, isolated in the CLI's own lockfile.
- Secret-safe argument contract: `parseCliArgs` uses `node:util` parseArgs strict with no `password`/`smtp-pass` option — both flags exit non-zero (verified). `--delay-ms abc` throws naming the flag rather than degrading to NaN.
- Runnable DRY-RUN driver: auto-detects the email column via `lib/core.detectEmailColumn` (with `--email-column` override), prints a `DRY RUN` banner and one `[i/N] would send -> <addr>  |  <merged subject>` line per row.
- Merge parity proven: `mergeRow` delegates to `lib/core.fillMessage` and is byte-identical to it, including the `$`-literal and spaces-in-column-key cases (SC-4).

## Decisions Made
- **Structural secret-safety over runtime checks:** the SC-2 guarantee is enforced by simply not declaring a `password`/`smtp-pass` option; strict parseArgs does the rejecting. No custom validation to bypass.
- **Import depth `../../../lib/core`:** corrected from the plan's `../../lib/core` (see Deviations).
- **Entrypoint guard on `main()`:** required so the parity test can import `mergeRow` without executing the CLI (which would `process.exit(1)` and kill the test runner).
- **DRY-RUN listing shows the merged subject** per row (beyond a bare address) to exercise `mergeRow` and make the preview substantive.

- **Fix:** Added a `readFileChecked(path, flag)` helper (resolve + `statSync().isFile()`, error names the flag) used for `--csv`.
