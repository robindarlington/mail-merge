## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| env / TTY → CLI | SMTP password enters ONLY via SMTP_PASS env or hidden prompt |
| CLI → user's SMTP server | nodemailer connection with explicit TLS mode |
| CLI → receipts file | Per-recipient outcome persisted to disk (no secrets) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-081-01 | Information disclosure | secrets.ts / argv | mitigate | Password only from SMTP_PASS env or promptHidden; no argv path exists (schema from Plan 01) |
| T-081-02 | Information disclosure | run.ts logs / receipts.ts | mitigate | No console.* of pass/auth; receipt entries exclude secret fields; capture-and-grep test asserts the pass string is absent from output + receipts (SC-2) |
| T-081-TLS | Tampering | createSmtpTransport secure flag | mitigate | Explicit `secure` boolean from SMTP_SECURE — never inferred from port (carry-forward fix) |
| T-081-DUP | (resume duplicate) | receipts.ts / run.ts | mitigate | fsync each receipt line before the next send; readSentSet skips recorded `sent` — documented as at-least-once (not exactly-once) per RESEARCH Pitfall 3 |
| T-081-03 | Tampering | --receipts path | mitigate | Resolve the receipts path; write only to the operator-supplied/derived path |
</threat_model>
