<objective>
Establish the `userId`-scoped data-access layer that every later phase inherits, plus the DTO redaction boundary that keeps the SMTP password off the wire. This is the tenancy + secrecy backbone of the phase: every function requires `userId` as its first parameter, the only shape that crosses to the client (`toSmtpConfigDto`) structurally omits the encrypted triple, and a single-row-per-user upsert is made race-safe by an additive UNIQUE index that is migrated onto disk.

Test-first: `lib/data/smtp.test.ts` (cross-tenant isolation, AUTH-02) and `lib/data/dto.test.ts` (redaction, SMTP-04) are Wave 0 scaffolds from RESEARCH Validation Architecture.

Purpose: AUTH-02 (multi-tenant isolation on every data access) and SMTP-04 (credentials encrypted at rest, password never returned to client).
Output: `lib/data/{smtp,index}.ts` + tests; `smtp_configs.user_id` UNIQUE index in `lib/db/schema.ts` + a committed Drizzle migration.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| request context (Clerk userId) → DAL | Only the server-derived userId may scope a query; never a client-supplied id |
| stored row → client response | The encrypted password triple must never cross to the client |
| concurrent upserts → single-row invariant | Two writes for the same user must not create two rows |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-2-IDOR | Information disclosure / Tampering | lib/data/smtp.ts queries | mitigate | Every function requires `userId` first param and filters on it; no by-id fetch without owner filter; cross-tenant test proves isolation (PITFALLS #13) |
| T-2-CRED | Information disclosure | toSmtpConfigDto boundary | mitigate | DTO explicitly picks safe fields; structurally omits password_enc/_iv/_tag; redaction test asserts a marker password is absent from JSON.stringify |
| T-2-DUPE | Tampering (data integrity) | upsert under concurrency | mitigate | Additive UNIQUE index on user_id makes onConflictDoUpdate atomic instead of a read-then-insert race |
| T-2-CRYPTO | Information disclosure | credential at rest | mitigate | Password persisted ONLY as the AES-256-GCM triple via lib/crypto encrypt() (no plaintext column exists) — SMTP-04 |
</threat_model>
