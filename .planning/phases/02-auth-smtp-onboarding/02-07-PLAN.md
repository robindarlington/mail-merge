<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| build stage → image layers | Anything ENV/ARG at build risks being inspectable in the image |
| host/Coolify secrets → running container | Runtime secrets must arrive at runtime, not bake into the image |
| public staging URL → app | The staging deployment is internet-reachable |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-2-BUILD | Information disclosure | Dockerfile build stage | mitigate | Only the client-safe `NEXT_PUBLIC_*` publishable key + URL vars are build ARGs; `CLERK_SECRET_KEY` and `CREDENTIAL_ENC_KEY` are runtime-only and never ARG/ENV in the Dockerfile (grep-asserted) |
| T-2-STAGING | Information disclosure | public staging URL on Clerk dev instance | accept | D-13 scopes staging to the Clerk dev instance (test keys, 100-user cap, dev banner); production instance + hardening are Phase 8; no real user data at risk on staging |
| T-2-VOL | Denial of service / data loss | /data SQLite volume | mitigate | Volume must be a LOCAL host volume (WAL requirement, never NFS) — verified in the deploy checkpoint; full persistence hardening is Phase 8 |
</threat_model>
