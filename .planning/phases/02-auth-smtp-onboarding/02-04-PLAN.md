**As a** signed-in user, **I want to** onboard and persist my own SMTP server proven functional by a live connection check, **so that** my credentials are stored encrypted at rest, reused across sessions, and never exposed to the client or logs.

| Clerk session → dashboard data fetch | The dashboard must fetch only the signed-in user's config |
| server row → client components | Only DTO fields may reach the client |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-2-IDOR | Information disclosure | dashboard config fetch | mitigate | Fetch via `getSmtpConfigForUser(await auth().userId)` — server-derived id only, never a client-supplied param |
| T-2-CRED | Information disclosure | dashboard → client props | mitigate | Pass only `toSmtpConfigDto` fields to client components; password triple never serialized into the RSC payload |
</threat_model>
