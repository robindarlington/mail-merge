## Phase Goal

**As a** signed-in user with an uploaded recipient list, **I want to** compose a plain-text email with merge-field autocomplete and save it as a template, **so that** I can reuse a correct, personalized message for my campaign.

<objective>
First user-visible vertical slice of Phase 4: a working `/compose` page where a signed-in user picks one of their saved recipient lists, composes a plain-text subject + body with `{{merge-field}}` autocomplete (click-to-insert chips + a `{{`-triggered Popover suggestion list), and saves the composed subject/body as a standalone template. After this plan a user can do something real they couldn't before: author and persist a personalized template. Preview + validation report land in Plan 05.

Purpose: EDIT-01 (compose subject+body), EDIT-02 (autocomplete/click-to-insert triggered on `{{`), EDIT-04 (save as template) — all wired end-to-end onto the Plan 03 `saveTemplate` action and the Plan 02 shared schema.

</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → saveTemplate action | Untrusted subject/body submitted for persistence (server re-validates via composeFormSchema — Plan 03) |
| CSV column names → rendered chips | Column names from columns_json are rendered as chip labels/inserted tokens |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-4-XSS-CHIP | Tampering | merge-field-menu chips | mitigate | column names rendered as JSX text (auto-escaped); inserted as literal `{{name}}` text, never as HTML |
| T-4-CLIENTVAL | Tampering | compose-editor submit | mitigate | client zod is UX only; the server (Plan 03) re-validates with the SAME composeFormSchema — never trust the client |
| T-4-IDOR | Information Disclosure | recipient-list Select | mitigate | page lists only listRecipientSetsForUser(userId); the client passes only recipientSetId to previewCampaign (Plan 05), never a path |
| T-4-SC | Tampering | shadcn add / npm installs | mitigate | textarea+popover are official shadcn blocks (registries:{}, no vetting gate); confirm `git diff package.json` adds NO dependency |
</threat_model>
