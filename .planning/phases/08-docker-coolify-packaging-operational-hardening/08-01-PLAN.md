## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| build args → image layers | Only client-safe NEXT_PUBLIC_* may cross; server secrets must not |
| host toolchain → runtime image | Dev deps (tsx/ts/drizzle-kit) must not reach the production runtime |
| container process → PID 1 | Signals must reach the node process directly, not a shell/npx wrapper |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-08-01 | Information Disclosure | secrets baked into image layers | mitigate | CLERK_SECRET_KEY/CREDENTIAL_ENC_KEY never a build ARG/ENV; verified with `docker history` (Task 2 verify) |
| T-08-02 | Elevation of Privilege | container runs as root | mitigate | `USER node` + node-owned /data in runtime stage |
| T-08-03 | Tampering | expanded attack surface via dev toolchain in prod | mitigate | pruned `npm ci --omit=dev` prod-deps stage replaces the full-node_modules copy |
| T-08-SC | Tampering | npm/pip/cargo installs | accept | Only esbuild promoted; already in lockfile transitively (`npm ls esbuild`), official project, no new download |
</threat_model>
