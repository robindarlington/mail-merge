Create the userId-scoped `campaigns` data-access layer — the durable unit both Phase-5 send surfaces operate against, and the home of the atomic draft→queued transition that makes TEST-03 double-submit-proof. The `campaigns` table already exists on disk (Phase 1 migration `drizzle/0000`); this plan adds NO schema change, only the DAL functions and their two-tenant tests.

Purpose: the single-statement `UPDATE ... WHERE status='draft'` enqueue guard is the true idempotency primitive for TEST-03; SQLite's single-writer model makes the affected-row count the authoritative "did I win the transition?" signal.
