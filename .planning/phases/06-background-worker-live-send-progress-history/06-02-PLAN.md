<objective>
Build the two per-recipient seams that are the correctness core of the phase: `materializeSendRecords` (turn a claimed campaign's CSV + template into one `pending` `send_record` per row, idempotently) and `runCampaign` (verify SMTP once, then walk the `pending` rows sending one personalized email each, committing every outcome immediately). This is where SEND-02/03/04/06 are realized.

Purpose: This is the phase's single hard correctness problem — send each recipient exactly once, survive a crash, never double-send, never leak the SMTP password. It reuses the already-tested `lib/core` primitives verbatim; no new merge/send/crypto/CSV code.
Output: `lib/worker/{materialize,process}.ts` + node:test seams (temp DB + injected stub transport, no live socket), green under `npm test`.
</objective>

## Phase Goal (user story)

**As a** signed-in sender, **I want to** have my queued campaign sent in a crash-safe background job with live progress and a durable per-recipient record, **so that** I can reliably deliver my whole batch, watch it happen, resume it after any crash without double-sending, and later review or download exactly what was sent.
