Route handler (06-RESEARCH.md lines 311-328, 06-PATTERNS.md lines 260-278):
  import { auth } from "@clerk/nextjs/server";   // DIRECT import here (not the lazy actions.ts workaround) — route handlers run only under the Next server runtime
  export async function GET(_req, { params }: { params: Promise<{ id: string }> })
  -> { userId } = await auth(); if !userId -> 401
  -> { id } = await params; campaign = getCampaignForUser(userId, Number(id)); if !campaign -> 404
  -> rows = getSendRecordsForCampaign(userId, campaign.id); csv = toResultsCsv(rows)
  -> Response(csv, { headers: Content-Type text/csv; Content-Disposition attachment; filename="campaign-{id}-results.csv" })

Formula-injection rule (06-RESEARCH.md line 237 / PITFALLS #12): prefix a field with a single quote (or tab) when its first char is one of = + - @ (also guard leading tab \t and CR \r). RFC-4180: wrap a field in double quotes and double any embedded quote when it contains a comma, double-quote, CR, or LF.
